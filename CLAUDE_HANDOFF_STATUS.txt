一、项目目标

这个项目目标是建立一个 FDA Advisory Committee briefing document 的监控、AI 分析、历史回测和纸面交易研究系统。

长期目标：
- 自动发现 FDA AdCom 新文件。
- 下载/解析 PDF。
- 用 AI 或规则分析 FDA briefing tone、efficacy、safety、questions framing、监管弹性。
- 生成正负面信号。
- 先做纸面交易和历史回测。
- 在足够验证后，未来才考虑实盘或 broker 接入。

当前明确不做：
- 不做自动实盘。
- 不接 broker。
- 不把 AI 输出直接当交易指令。

四、有无调取 API 分析

代码层面已经支持 AI API：
- OpenAI
- Anthropic
- DeepSeek
- heuristic fallback

但是本轮我没有新增大批量调用 AI API。

原因：
1. 当前更大的瓶颈不是调用 API，而是 outcome 标签缺失。
2. 如果 outcome 没有 source，批量 AI 分析无法形成可靠 backtest。
3. 目前 labeled_analyzed_count 只有 8，且几乎全是 positive/approved，模型评估会严重偏。
4. 用户要求不能编造 outcome，所以我优先做审计、结构升级和安全标注。

历史上已有部分 JSON 是 AI 分析结果：

但这不是本轮新 API 批量调用。

- 当前很多数据只有 meeting_date。

七、离实盘还有多远

当前不适合实盘。

保守判断：
- 至少还需要 1-3 个月 paper trading。
- 至少需要 30-50 个高质量 verified historical labels。
- 最好需要 100+ 个多类型历史样本，包括足够负面和争议案例。
- 需要在新 prompt + 新 signal engine 下重跑 backtest。
- 需要 paper trading 连续稳定，再考虑小仓位手动交易。

当前适合阶段：
- Research dashboard
- Historical labeling
- Prompt/signal iteration
- Paper trading

不适合阶段：
- Broker integration
- Automatic live trading
